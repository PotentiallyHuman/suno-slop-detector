v0.3.0's headline feature is a NON-ENGLISH GUARD. The model is English-only (its BoW + cliché
lexicon are English keywords), so non-English lyrics (e.g. Danish) get a meaningless low score —
a Danish line has only ~3/32 tokens recognized, so the model is effectively blind and the number
is noise. v0.3.0 must DETECT non-English on-device (NO network, NO API, NO LLM) and, instead of a
misleading number, show a clear notice with the guidance routed through the joker card.

This exact behavior already exists in the repo's standalone app at app/app.js — copy its approach.
The detection is a pure function-word ratio (real English ≈30-40% common function words;
Danish/Spanish ≈0-6%):

  // <8% common English function words ⇒ not English (real English never drops near this)
  var EN_COMMON = /^(the|and|you|to|a|of|in|it|that|is|my|me|we|for|on|with|but|love|night|i|are|was|be|he|she|they|this|have|not|your|all|like|when|what|so|do|can|just|know|now|time|up|out|no|yes|oh|don't|i'm|we're|you're|it's)$/;
  function looksNonEnglish(text){
    var toks = String(text).toLowerCase().match(/[a-z']+/g) || [];
    if (toks.length < 12) return false;            // too short to judge
    var hits = 0; for (var i=0;i<toks.length;i++) if (EN_COMMON.test(toks[i])) hits++;
    return (hits / toks.length) < 0.08;
  }
